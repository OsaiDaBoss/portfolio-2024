Restaurant Website

This project consists of a website with a physical architecture that allows users to interact with it. With the click of a button, the user can give commands such as showing all products, showing a product by its ID, adding a product, editing a product, or deleting a product. These commands are communicated from the frontend to the backend, which interacts with a Mongo database to retrieve, create, update, or delete data. The backend then communicates the changes back to the frontend, which displays the updated information on the website. The website offers three views for user interaction. The Menu view allows users to see all the products or search for a specific product by its ID using a search bar. The searched product is displayed prominently with its description, while the other products are listed below. Any changes made in the DevMode view are reflected in the Menu view. The DevMode view is intended for developers and allows them to edit the database. They can add new products, modify existing products, or delete products. The changes made in this view are directly reflected in the database, which in turn updates the Menu view on the website.
